How PhishGuard Works
1
Enter a URL
Paste any suspicious link into the URL Analyzer. PhishGuard accepts any web address — shortened URLs are automatically expanded before analysis.
2
Rule-Based Analysis
15+ security rules instantly check for red flags — IP-based domains, suspicious TLDs (.tk, .ml, .xyz), brand impersonation, missing HTTPS, excessive subdomains, and more.
3
Machine Learning Detection
A Random Forest classifier (150 decision trees, trained on 500K+ URLs) extracts 35 structural features from the URL and predicts phishing probability with 90.94% accuracy.
4
Domain Intelligence
WHOIS lookup checks domain age and expiry. SSL certificate is verified. Blacklist APIs (PhishTank, Google Safe Browsing) cross-reference known phishing databases.
5
Combined Verdict
All signals are merged into a single risk score. You get a clear Safe / Suspicious / Phishing verdict with a detailed breakdown of every flagged issue.
What We Detect
⚠
Brand Impersonation
Fake PayPal, Google, Amazon, bank domains using typosquatting or lookalike characters
🔗
URL Shorteners
bit.ly, tinyurl, and 50+ shortener services that hide the real destination
🌐
Suspicious TLDs
High-risk extensions (.tk, .ml, .ga, .cf, .xyz) commonly used in phishing campaigns
📡
IP-Based URLs
Direct IP address URLs (e.g. http://192.168.1.1/login) that bypass DNS entirely
🔒
Missing HTTPS / SSL
Unencrypted HTTP connections and invalid or missing SSL certificates
🕵
Obfuscation Techniques
Encoded characters, excessive subdomains, redirect chains, and homograph attacks

About PhishGuard

Our Mission in Cybersecurity

We build tools that empower everyone to browse the web safely, without needing a degree in cybersecurity.

Mission

Making the Web Safer, One URL at a Time

PhishGuard is a final year project built by a team of three cybersecurity students at Hamdard University. Our goal is to create an accessible, accurate, and real-time phishing detection system that anyone can use. By combining rule-based analysis with a powerful machine learning model, we provide immediate feedback on whether a URL is safe or malicious — helping users avoid phishing attacks before they click.

How It Works

Dual-Layer Detection

  • Layer 1 — Rule Engine: Checks URLs against 15+ security rules (suspicious TLDs, brand impersonation, IP addresses, special characters, URL shorteners, etc.)
  • Layer 2 — ML Model: A Random Forest classifier (150 decision trees) trained on 500K+ URLs extracts 35 structural features and predicts phishing probability
  • Combined Verdict: Both scores are merged to produce a final risk assessment with clear visual feedback

Performance

Model Accuracy

  • Accuracy: 90.94% on test data
  • Dataset: 507,196 URLs after deduplication
  • Features: 35 URL-based features
  • Top Predictors: Digit count, sensitive keywords, dot count, TLD patterns
  • Split: 80% training / 20% testing
