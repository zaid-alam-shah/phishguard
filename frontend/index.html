How PhishGuard Works
1
Enter a URL
Paste any suspicious link into the URL Analyzer. PhishGuard accepts any web address — shortened URLs are automatically expanded before analysis.
2
Rule-Based Analysis
15+ security rules instantly check for red flags — IP-based domains, suspicious TLDs (.tk, .ml, .xyz), brand impersonation, missing HTTPS, excessive subdomains, and more.
3
Machine Learning Detection
A Random Forest classifier (150 decision trees, trained on 500K+ URLs) extracts 35 structural features from the URL and predicts phishing probability with 90.94% accuracy.
4
Domain Intelligence
WHOIS lookup checks domain age and expiry. SSL certificate is verified. Blacklist APIs (PhishTank, Google Safe Browsing) cross-reference known phishing databases.
5
Combined Verdict
All signals are merged into a single risk score. You get a clear Safe / Suspicious / Phishing verdict with a detailed breakdown of every flagged issue.
What We Detect
⚠
Brand Impersonation
Fake PayPal, Google, Amazon, bank domains using typosquatting or lookalike characters
🔗
URL Shorteners
bit.ly, tinyurl, and 50+ shortener services that hide the real destination
🌐
Suspicious TLDs
High-risk extensions (.tk, .ml, .ga, .cf, .xyz) commonly used in phishing campaigns
📡
IP-Based URLs
Direct IP address URLs (e.g. http://192.168.1.1/login) that bypass DNS entirely
🔒
Missing HTTPS / SSL
Unencrypted HTTP connections and invalid or missing SSL certificates
🕵
Obfuscation Techniques
Encoded characters, excessive subdomains, redirect chains, and homograph attacks
